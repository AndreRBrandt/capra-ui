# ADR-018: Design Token Enforcement

## Status

Aceito

## Contexto

Após auditoria do `DashboardPage.vue`, foram encontradas múltiplas violações:
- Cores hex hardcoded (`#10b981`, `#ef4444`, `#e5a22f`, `#374151`, `#e5e7eb`) em inline styles
- `style="background: #fef3c7; color: #92400e;"` no template para legendas do heatmap
- Hex literals espalhados em formatters de chart (ECharts/Chart.js recebem strings HTML)

Essas violações criam dois problemas:
1. **Dark mode quebra** — cores fixas não respondem a troca de tema via `data-theme`
2. **Identidade visual requer busca em N arquivos** em vez de alterar 1 token

## Decisão

**Zero hex hardcoded em templates Vue. Zero inline styles em páginas.**

### Regras concretas

| Contexto | Proibido | Obrigatório |
|----------|----------|-------------|
| Template Vue (cor de texto/bg) | `style="color: #ef4444"` | Classes CSS com `var(--color-*)` |
| Prop de componente (accent-color) | `accent-color="#9b2c2c"` | `KPI_COLORS.discount` |
| Formatter de chart (string HTML) | `"color: #10b981"` | `\`color: ${CHART_COLORS.positive}\`` |
| Nova cor KPI | Constante local `const C = '#...'` | Adicionar em `tokens.css` + `KPI_COLORS` |

### Estrutura de tokens

```
capra-ui/src/styles/tokens.css          ← fonte canônica de todas as cores
  :root {
    --color-kpi-*                        ← categorias de KPI
    --color-heatmap-*                    ← escala de intensidade de heatmap
    --color-trend-*                      ← positivo/negativo
    --color-brand-*                      ← identidade da marca
    ...
  }

capra-analytics/src/constants.ts
  KPI_COLORS   ← mapeia para var(--color-kpi-*), usar em accent-color de KpiCard
  CHART_COLORS ← hex mirror dos tokens, usar APENAS em formatters de chart
```

### Exceções aceitas: `CHART_COLORS` e `KPI_CHART_PRESETS`

Bibliotecas de chart (ECharts, Chart.js) recebem strings HTML onde CSS custom properties não
são interpoladas. Para esses casos, dois objetos em `constants.ts` são as **únicas exceções**:

1. `CHART_COLORS` — objeto JS com hex que **espelham** os tokens CSS. Usar em formatters que
   constroem strings HTML (tooltips, labels). Ao alterar, atualizar `tokens.css` **E** `CHART_COLORS`.

2. `KPI_CHART_PRESETS` — array readonly com paleta de cores para séries ECharts (chart config).
   Hex nunca aparece fora desses dois objetos/constantes no app.

## Consequências

### Positivas
- Dark mode funciona automaticamente em toda UI
- Mudanças de identidade visual alterando apenas `tokens.css`
- Code review mais fácil: violation = hex literal no template

### Negativas
- `CHART_COLORS` é um espelho manual — pode desincronizar se tokens CSS mudarem sem atualizar o objeto JS
- Requer disciplina no review (adicionar ao checklist de AP-15)

## Alternativas Consideradas

- **CSS-in-JS / styled components**: rejeitado — projeto usa Tailwind v4 + CSS vars, adicionar outra camada seria over-engineering
- **PostCSS plugin para extrair vars**: rejeitado — complexity não justificada para o tamanho atual do projeto
